# Лабораторная работа № 5

## Тема

Алгоритмы отсечения отрезка произвольным многоугольником в двумерном пространстве.

## Вариант

- **Алгоритм отсечения:** отрезка произвольным многоугольником
- **Размерность пространства отсечения:** двумерное (2D)
- **Тип отсечения:** отсечение отрезка (внутренняя область)

## Задание

### 1. Основной алгоритм отсечения

Реализовать алгоритм отсечения произвольного отрезка выпуклым или вогнутым многоугольником:

- Нахождение всех точек пересечения отрезка с рёбрами многоугольника.
- Определение видимых фрагментов отрезка, лежащих внутри многоугольника (правило чётного-нечётного).
- Визуализация отсечённых частей отрезка отличным от контура цветом.

### 2. Вспомогательные алгоритмы

- Построение и заливка многоугольника (алгоритм растровой развертки **A5** из [лабораторной работы №4](../lab4)).
- Постфильтрация изображения с взвешенным усреднением области \(3 \times 3\) (алгоритм **Б2** также из предыдущей [лабы 4](../lab4)).
- Растеризация отрезков (алгоритм Брезенхэма) для отрисовки контура и видимых фрагментов.

### 3. Интерактивный ввод данных

Предусмотреть интерактивный ввод исходных данных с помощью мыши и клавиатуры:

- Построение многоугольника: левая кнопка мыши добавляет вершину, правая кнопка (или клавиша `Enter`) замыкает контур и переключает режим в **отсечение**.
- Ввод отрезков: левый клик задаёт начало отрезка, второй клик — конец. После ввода выполняется отсечение, и видимые фрагменты добавляются на сцену.
- Предварительный просмотр вводимого отрезка (динамическая линия до курсора) как при построении многоугольника, так и при вводе отрезков.

### 4. Управление и дополнительные возможности

- **Очистка сцены:** клавиша `C` полностью сбрасывает состояние (многоугольник, отрезки, режим).
- **Отмена последнего действия:** клавиша `Backspace` удаляет последнюю добавленную вершину (если многоугольник не замкнут) или отменяет начатую точку отрезка.
- **Выход из приложения:** клавиша `Esc`.
- **Изменение размеров окна:** корректно обрабатывается пересозданием буферов.

### 5. Растеризация и вывод

Растеризация выполняется в программном буфере в памяти с последующим копированием результата в буфер кадра OpenGL с помощью `glDrawPixels`. При изменении размеров окна буферы пересоздаются с сохранением масштаба изображения.

## Примечание

Программа продолжает архитектуру [лабораторной работы №4](../lab4) и расширяет её функциональность возможностью интерактивного отсечения отрезков.  
Алгоритм отсечения работает с произвольным многоугольником и правильно обрабатывает все случаи, включая касание отрезка вершин или рёбер.
